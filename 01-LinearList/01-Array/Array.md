# 数组
## 什么是数组
**数组是一种线性表数据结构，它用一组连续的存储空间来存储一组相同类型的数据**。怎么样，听起来是不是很简单？在我们还没有接触数据结构与算法之前，相比就已经对数组十分熟悉了。但是在今天，我们来重新认识一下这位“老朋友”。
## 数据表结构与连续的存储空间的优势：数组的随机访问
我们知道，计算机会给每个内存单元分配一个地址，并且通过这个地址去访问他们的值。数组的数据表结构与连续的存储空间两大限制结合起来，让计算机对数组的元素访问变得十分简单。对于任意arr[i]，计算机会去访问**address=arr_address+i*data_type_size**处的地址，并从中获取它们所对应的值。这样的随机访问对应的时间复杂度是O(1)。顺带一提，数组虽然适用于数据查找，但是数据查找的时间复杂度并不是O(1)，如果是有序数组二分查找，时间复杂度为O(logn)，如果是无序数组，时间复杂度为O(n)。  
这里说句题外话：刚学数组的同学可能会很疑惑，为什么数组的下标是从0开始而不是从1开始呢，反人类呀！但是，你看到上面的那个等式便会茅塞顿开了。数组的优势是什么？就是**随机访问**呀！如果设计师决定让数组从1开始，那么计算机要访问的地址就会是：address=arr_address+i*data_type_size **-data_type_size**。也就是说，数组的每次随机访问就会多计算一次减法，一次两次的时间可以忽略不计，但是对于大型项目来说会是一段很长的时间开销。
## 数据表结构与连续的存储空间的缺陷：数组低效的插入与删除
我们知道，数组的结构是严密的、连续的，在这样的线性结构下，删除数组最后一个数据、增添数据到数组的最末尾是很轻松的。但是对于数据的插入与中间元素的删除，无疑会形成天然的排斥。举个例子：如果我们要把数据a插入数组arr的第k个位置（k<arr.size()）会发生什么？我们需要把原来第k个位置的数据搬到k+1的位置，把原来第k+1个位置的数据搬到第k+2个位置……如此，才能在原来紧密的排布里挪出一个“空位”。这听起来真是十分消耗心神呀！对于时间的消耗也是十分巨大，数据插入与删除的时间复杂度为O(n)，在大型数组中可以说是**十分低效**了！
## 警惕数组的越界
这是学生们在初学数组时老生常谈的问题，在这里比较坑的点是：程序不会报错。这是因为计算机确实能够访问该地址，哪怕它越界了，所以在我们使用数组时，要关注数组是否越界这一难点。
## 数组的扩容
既然要警惕数组的越界，但是数组的空间毕竟是有限的、静态的。在后续的程序中数组就是会越界，这可怎么办？这就需要数组的扩容。  
如何扩容？这就需要用到**动态数组**的知识了，代码如下：
```cpp
#include <iostream>
using namespace std;
int main(){
    // 初始，创建一个大小为3的数组
    int* arr=new int[3];
    //记录容量
    int room=3;
    //记录输入数据的个数
    int count=0;
    int num;
    while(cin>>num){
        arr[count++]=num;
        //如果输入数据个数与容量相等，需要扩容！
        if(count==room){
            //新容量为原来的两倍（一般情况下）
            room*=2;
            int* new_arr=new int[room];
            //将原来的数据复制到新数组上
            for(int i=0;i<count;i++){
                new_arr[i]=arr[i];
            }
            //释放空间
            delete[]arr;
            //重新命名，利于进入下一次循环
            arr=new_arr;
        }
    }
    //检测结果
    for(int i=0;i<count;i++){
        cout<<arr[i]<<" ";
    }
    //释放内存
    delete[] arr;
    return 0;
}
```
为什么这里一开始使用int* arr=new int[3]而不使用int arr[3]？这是因为int arr[3]定义的是一个静态数组，其中的arr是一个常量指针，指向固定内存，不能被赋值，在后续的arr=new_arr步骤时会报错，因为常量不能修改。
